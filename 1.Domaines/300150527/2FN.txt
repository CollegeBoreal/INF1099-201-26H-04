2NF – Deuxième Forme Normale

Pour atteindre la 2NF, la table ORDER_DETAILS a été décomposée
afin d’éliminer les dépendances partielles.

Les nouvelles tables sont :

TABLE : USER
- userId (PK)
- userName
- userPhone
- userAddress

TABLE : PRODUCT
- productId (PK)
- productName
- productPrice

TABLE : ORDER
- orderId (PK)
- orderDate
- userId (FK)
- paymentMethod

TABLE : ORDER_ITEM
- orderId (FK)
- productId (FK)
- quantity
PRIMARY KEY (orderId, productId)

Dans cette forme :
- Tous les attributs non clés dépendent entièrement de la clé primaire
- Il n’existe plus de dépendance partielle
